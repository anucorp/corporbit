1. Use the Maven wrapper

Maven Wrapper controls the Maven version used by a project. It allows a developer to build a project using the correct Maven version without a local Maven install. It has been accepted to be included in the Apache Maven project itself, and will likely be a feature in the upcoming Maven 4.0.0 release.

1.1. Setup the Maven wrapper

To setup the Maven Wrapper, run the following command in the root of your project:

mvn -N io.takari:maven:0.7.7:wrapper -Dmaven=3.8.3

To use the wrapper, just replace your regular mvn command with ./mvnw (or ./mvnw.cmd). E.g.:

./mvnw clean install -DskipTests

This will download and use the specified Maven version.

1.2. Change Maven version

To change the Maven version used by the project, run the command

./mvnw -N io.takari:maven:0.7.7:wrapper -Dmaven=3.6.3

using the desired Maven version, and check in any changed mvnw, mvnw.cmd, or .mvn/wrapper files.

1.3. Faster maven builds

To build using maven the default build command is

./mvnw clean install -DskipTests

A full rebuild can take over 15 minutes depending on the hardware used.

Fast building when working on backend / Configuration studio
mvn clean install -pl bmp-backend,app-bmp-config-studio -am -DskipTests

This will build backend and confgiguration studio without building the frontend (takes long time) After this it wont be possible to run the frontend, but CS and Server should start fine.

Fast building a change in the bmp-api module

The bmp-api module has some issues where any code in it changes it needs a full maven rebuild.

This is how to make it faster, first build the bmp-api module (only) assuming bmp-frontend has already been built.

mvn install -pl bmp-api -DskipTests

Then copy over the classes to the bmp-frontend 

cp -r ./bmp-api/target/bmp-api/WEB-INF/classes/com/corporater/bmp/api/ ./bmp-frontend/target/bmp-frontend/WEB-INF/classes/com/corporater/bmp/

Then rebuild in intellij to have the changes applied

Note : This copy command will also copy files from bmp-api into frontend after compiling using intellij. So the maven command above is not needed when working iteratively on bmp-api code. Just the cp command is enough.

linux aliases to make this simpler (insert into .zshrc or .bashrc)
alias compileapi="mvn compile -pl bmp-api -DskipTests "
alias mvncbe='mvn clean install -pl bmp-backend,app-bmp-config-studio -am -DskipTests'
alias cpapi="cp  -r ./bmp-api/target/bmp-api/WEB-INF/classes/com/corporater/bmp/api/ ./bmp-frontend/target/bmp-frontend/WEB-INF/classes/com/corporater/bmp/"
alias mvnapi="compileapi ; cpapi"

Now the mvnapi command can be used to build changes in bmp-api
