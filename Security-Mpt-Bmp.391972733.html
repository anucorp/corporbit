Security Mpt Bmp

Security BMP Manual Penetration Test

Setup

Important notice: * Create a non-admin user without access to ConfigStudio as a user for performing MPT with a strong password. * Change admin password to secure password. After each application update, admin password is reset to default. Remember to always set a secure admin password after update.

Server Configuration

  • Application Server
  • Use port 443
  • Disable "Show errors on web"

  • Logging (keep defaults)

  • Mail

  • Configure mail (individual setup, no predefined values)

  • SSO (disabled)

  • Login/Logout

  • Failed attempts allowed: 3
  • Lockout period length: 15
  • Enable "Only one session per user"

  • Password Policy
  • Password validity: 90
  • Password length: 12
  • Number of new passwords required: 5
  • Enable "Lower case characters required"
  • Enable "Upper case characters required"
  • Enable "Special characters required"
  • Enable "Numbers required"

  • SSL
  • Enable SSL
  • Enable HSTS

  • Sessions
  • Inactive Session Time, Max: 300
  • Number of Sessions, Max: 1
  • Enable "Show session event log"

  • Security
  • SameSite Cookie: Strict
  • Disable all checkboxes
  • Valid Extension (for File upload): pdf, doc, docx, xls, xlsx, ppt, pps, pptx, ppsx, csv, txt
  • Enable Clam antivirus

  • Workspaces (Workspace Defaults)
  • Disable "Scheduler"

  • Workspace & Data Configuration: default
  • Disable "Scheduler"
  • Preferably use database instead of embedded

  • Edit GraphQL requests and check if payload is returned to web as part of error message.

  • Inject \<script>alert(‘1’)\</script> or html code in all possible places (search, comment filed, path, etc.).

  • /CorpoWebserver/web/i18n?lang=en
  • /CorpoWebserver/graphql
  • /CorpoWebserver/web/uploaded/temp?fileName=&dirName=
  • /CorpoWebserver/web/js/vendor/.htaccess
  • /CorpoWebserver/web/js/bundle/.htaccess

This test can be automized.

  • Test session fixation (session id before login and after login should not be the same).

  • Test uploading files (both with allowed and forbidden extensions):

  • Images on web
  • Attachments on web
  • Images from Postman

  • Attachments from Postman

  • Test edition of user password via GraphQL query manipulation.

  • Test unauthorized access to organizations.
