Junit5 Migration

Migrating existing tests from JUnit 4 to JUnit 5

If JUnit 5 has not been added to the module that you are working on, it needs to be included in pom.xml (here also a version of Mockito compatible with v.5):

<dependency> <groupId>org.junit.jupiter</groupId> <artifactId>junit-jupiter</artifactId> </dependency> <dependency> <groupId>org.mockito</groupId> <artifactId>mockito-junit-jupiter</artifactId> </dependency>

In IntelliJ IDEA:

  1. Install JUnit 4 to 5 Converter plugin.
  2. Enable JUnit 4 test can be JUnit 5 inspection:
    Editor > Inspections > JVM Languages: JUnit 4 test can be JUnit 5
  3. Pick a test suite which can be converted to JUnit 5 and execute the Convert to JUnit 5 action, which will handle part of the job for you.
  4. Run the tests.
  5. Use error logs and migration guides to fix any errors which appeared after conversion, e.g.
    https://medium.com/@wenqiglantz/junit-migration-guide-from-junit-4-to-junit-5-95b5700431a6

Pro tips:

  1. Test classes and test cases no longer have to be public in JUnit 5.
  2. @RunWith has been replaced with @ExtendWith.
  3. MockitoJUnitRunner has been replaced with MockitoExtension.
  4. @DisplayName annotation can be used to provide an easy-to-read description of test suites and test cases.
  5. JUnit 5 offers an out-of-the-box support for parameterized tests, which makes them really easy to write.
    See: https://www.baeldung.com/parameterized-tests-junit-5

NOTE

It is not necessary to migrate all the existing tests from JUnit 4 to JUnit 5, but all new tests should be written with JUnit 5.
