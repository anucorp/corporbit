Corporater Developer Documentation

Welcome to the Corporater Developer Documentation.

This is a place where we can keep internal documentation for developers. It's modelled after the Awesome Lists on GitHub, and should be a place to collect information about teams, projects, best-practices, things that interests us, tips & tricks, etc. In short, just about anything.

Our main objective is to collect documentation important to us, the developers of Corporater, in one place! Currently, a lot of documentation and know-how is spread across the wikis, Teams, and other places, making it hard to find if you don't know where to look. Please help your fellow developers by listing it here.

Guidelines

  • All developers are encouraged to create and post documentation as GitHub markdown files[^1].

  • You can add just about any kind of content; list of interesting blog articles, links to external documentation, collections of tips and tricks, etc.

  • This is a beta project, folder structure is of free choice.

  • If you create a new folder, please include a README.md file, preferably with a table of contents. This makes it a lot easier to navigate the structure, since the README.md file will be automatically displayed when browsing the repository on GitHub.

  • If you have a team or project wiki, please consider moving it here! It's quite easy to clone the wiki pages, and then copy them to a suitable place in this repository. At the very least, make a page in this repository with the table of contents of your wiki, linking your content.
